# Docs Index

This folder is organized by purpose so analysis, implementation, and reporting can evolve independently.

## Structure

- `analysis/` competition framing, solver options, and strategy tradeoffs
- `architecture/` system design, data model, and module boundaries
- `handoff/` session continuity notes, status, and next-session materials
- `implementation/` roadmap, experiments, and delivery plan
- `report/` technical report assets and writing structure
- `presentation/` pitch and demo framing

## Recommended Reading Order

1. `analysis/problem-framing.md`
2. `analysis/strategy-options.md`
3. `architecture/system-design.md`
4. `architecture/data-model.md`
5. `handoff/current-status.md`
6. `handoff/checklist.md`
7. `implementation/roadmap.md`
8. `report/technical-report-outline.md`
