# Competition Plan

## Before Preliminary Opens
- finish parser and feasibility checker
- create synthetic instances for fast testing
- implement constructive baseline and logging

## Early Preliminary
- submit stable feasible solutions first
- collect leaderboard signal before optimizing aggressively
- identify which instance families seem hardest

## Mid Preliminary
- add adaptive neighborhoods and local exact repair
- run ablations to keep only features that pay off
- tune time allocation across constructor and search

## Late Preliminary
- harden against edge cases
- use final submissions for robustness, not speculative changes
- freeze documentation and experiment artifacts

## Final Round Preparation
- turn logs into clear report figures
- prepare explainable examples of retrieval-aware behavior
- polish code structure for open-source disclosure
